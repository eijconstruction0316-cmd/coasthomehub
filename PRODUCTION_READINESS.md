## Operational Notes

The current rate limiter is a local in-memory guard. It is useful for a single server process and local production safety, but it is not a distributed abuse-control system.

The quote bot protection is server-side only and does not alter the UI. It blocks unexpected honeypot fields and cross-origin production submissions. A stronger future control should add an invisible challenge provider.

The report submission flow now requires a short-lived signed report token generated by `/api/generate-report`. This prevents clients from bypassing report generation and directly submitting arbitrary report HTML to email templates.

Checkout success UI is still not an authorization boundary. Paid access should eventually be backed by persisted Stripe subscription state rather than localStorage unlock flags.
